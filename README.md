# Malware Detection Using Control Flow Graph Topology Features with Classical Machine Learning

> **Paper:** "Malware Detection Using Control Flow Graph Topology Features with Classical Machine Learning"  
> **Author:** Anushka Srivastava — VIT Chennai (25BCE5388)

## Overview

This repository demonstrates that **16 hand-crafted CFG topology features** combined with a **CPU-only ExtraTrees classifier** can match GPU-dependent Graph Neural Network approaches within a narrow performance gap — while remaining fully interpretable and lightweight.

### Key Results

| Metric | Our Method (ExtraTrees) | Gao et al. 2022 (GNN) |
|--------|------------------------|----------------------|
| F1 Score | 97.67% ± 1.21% | 99.19% |
| Accuracy | 97.70% ± 1.19% | — |
| AUC | 0.9965 | — |
| Hardware | CPU only | GPU required |
| Interpretable | ✓ (16 named features) | ✗ (384D embeddings) |

**Gap vs. baseline: 1.52 percentage points** — well within the ≤5 pp hypothesis threshold.

## Dataset

1,000 balanced PE binary samples replicating Gao et al.'s MGD-BINARY composition:

- **500 malware** — 8 BODMAS families: sfone, upatre, wabot, sillyp2p, musecador, padodor, gandcrab, dinwod
- **500 benign** — DikeDataset (same source as Gao et al.)

CFGs extracted from real PE binaries using **angr's CFGFast** static analysis engine. BODMAS samples restored from disarmed state via `PE_modifier.py`.

> **Note:** `sillyp2p` replaces Gao et al.'s `benjamin` family. All benjamin samples produced zero-node CFGs due to packing — a documented static analysis limitation, not a data error.

## Features (16 CFG Topology Features)

| Category | Features |
|----------|----------|
| Basic | nodes, edges |
| Complexity | cyclomatic_complexity, weakly_connected_components, scc_count |
| Density | density |
| Centrality | avg_betweenness, max_betweenness |
| Degree | max_in_degree, max_out_degree |
| Structural | leaf_nodes, root_nodes |
| Engineered Ratios | complexity_per_node, edge_to_node_ratio, scc_to_node_ratio, leaf_ratio |

`avg_in_degree` and `avg_out_degree` were dropped as they are mathematically identical to `edge_to_node_ratio` in directed graphs.

## What the Script Does

1. **Dataset summary** — composition, family distribution, balance check
2. **Model comparison** — 10 configurations (4 vanilla + 5 stacking + 1 voting) evaluated via 10-fold stratified CV
3. **Statistical significance** — paired t-tests confirming ExtraTrees significantly outperforms all alternatives (p < 0.025)
4. **Final model evaluation** — ExtraTrees with full metrics, ROC curve, confusion matrix
5. **Feature importance** — ranked feature contributions with category labels
6. **Feature direction analysis** — malware vs. benign structural signatures
7. **Hypothesis test** — formal comparison against Gao et al. (2022)

## Key Findings

- **Malware signature = fragmentation + dead-ends.** Weakly connected components (~4.4× higher in malware) and leaf ratio (~1.4× higher) are the top discriminators.
- **Benign binaries are structurally larger and denser** — a counterintuitive but verified finding.
- **Betweenness centrality is the least important feature** despite theoretical promise — both classes show similar centrality distributions.
- **ExtraTrees significantly outperforms all 9 alternative configurations** including stacking ensembles, at α = 0.025.
